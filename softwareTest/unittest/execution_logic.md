
# unittest 的执行逻辑

`unittest` 的执行逻辑如下：
1. **测试用例发现**：
   - 执行 `unittest.main()` 时，框架会扫描当前模块中的测试类和测试方法。
   - 测试类必须继承自 `unittest.TestCase`，测试方法名必须以 `test_` 开头。

2. **运行测试用例**：
   - 框架调用测试类中的每个测试方法，按照定义的顺序依次运行。
   - 每个测试方法会单独运行，且会在运行前后执行 `setUp` 和 `tearDown` 方法。

3. **记录和报告结果**：
   - 框架会记录测试结果，标记为通过、失败或错误。
   - 测试运行器会将结果输出到控制台或生成详细报告。
